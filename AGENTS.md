# Repository Working Instructions

These instructions capture the owner's preferred workflow for future Codex sessions.

- Develop new features locally first.
- Run and inspect the site locally before considering any push. Use the local browser when available, or ask the owner to verify in their browser.
- Never push directly to `main` unless the owner explicitly approves that exact push in the current conversation.
- Treat `main` as the release branch. Prepare and verify changes locally, then ask for approval before every push to `main`, even for small documentation-only changes.
- If there is any doubt about whether approval was granted, stop and ask before pushing.
- If browser verification is not possible, say so clearly and describe what was checked instead.
- Do not revert or overwrite the owner's local changes unless the owner explicitly asks for that.
